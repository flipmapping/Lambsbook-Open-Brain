EVIDENCE:
Multiple MVP convergence investigations encountered periods where execution could not proceed because required authority, evidence, access, runtime state, schema visibility, or environmental conditions were unavailable.

Progress resumed most effectively when recovery was structured around verified evidence and a single executable recovery path.

TRUTH:
When execution cannot proceed, recovery must be driven by evidence rather than speculation.

MEANING:
Blocked execution is not a failure state.

Blocked execution is a truth-discovery state.

The objective becomes localization of the blocker and identification of the smallest verified recovery path.

RULE:
When execution is blocked:

1. State the blocker.

2. State the verified evidence.

3. State the recovery path.

4. State the exact artifact required.

5. State the single next executable step.

Never substitute assumptions for missing evidence.
