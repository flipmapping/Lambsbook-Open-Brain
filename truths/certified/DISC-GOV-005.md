EVIDENCE:
Knowledge systems frequently accumulate object types faster than they accumulate useful knowledge.

Additional object classes increase maintenance cost, capture complexity, retrieval complexity, and governance burden.

The value of a semantic object type must exceed the complexity introduced by its existence.

TRUTH:
New semantic object classes should be admitted only when they enable decisions that cannot be reliably supported by existing object classes.

MEANING:
Ontology growth must be justified by decision utility rather than conceptual distinction alone.

RULE:
Before creating a new object class, answer:

"What decision can be made with this object that cannot be made reliably with an existing object?"

If no clear answer exists, do not create the object class.
