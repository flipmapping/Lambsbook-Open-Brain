# LESSON-EXEC-002

Title:
Instruction Block ≠ Executable Block

Authority:
OPEN-BRAIN-012

Domain:
Execution

Pattern

Human-readable recovery plans must remain separate from machine-executable next steps.

Observed Failure Mode

When narrative recovery guidance and executable commands are combined into a single block:

- execution intent becomes ambiguous
- copy-paste reliability decreases
- automation compatibility decreases
- agent orchestration becomes more difficult

Lesson

Instruction blocks and executable blocks serve different purposes.

Instruction Block

Purpose:
Explain

Audience:
Human

Contents:
- reasoning
- context
- recovery path
- decision rationale

Executable Block

Purpose:
Execute

Audience:
Human or Agent

Contents:
- commands only
- no narrative
- no interpretation
- deterministic action

Operational Rule

Never mix recovery narrative with executable commands.

Present:

1. Recovery Guidance
2. Executable Artifact

as separate artifacts.

Strategic Significance

This distinction becomes increasingly important as Open Brain evolves toward:

- ontology-aware execution
- agent orchestration
- autonomous recovery workflows
- machine-assisted governance
