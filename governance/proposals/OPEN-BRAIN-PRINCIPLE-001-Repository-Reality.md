# OPEN-BRAIN-PRINCIPLE-001 — Repository Reality

Status: Proposed
Founder Approval: Approved

## Purpose

Establish Repository Reality as a foundational architectural principle of
the Open Brain platform.

This principle defines the relationship between architectural intent,
repository state, and implementation truth.

---

## Repository Reality Principle

The repository is the sole authoritative representation of the currently
implemented system.

Architectural decisions, historical conversations, candidate artifacts,
implementation plans, remembered work packages, prior analyses, and
implementation proposals SHALL NOT be used to infer repository state.

Every implementation cycle SHALL begin with Repository Certification.

Execution Briefs SHALL be derived from certified repository facts together
with approved architectural authority.

When architectural intent and repository reality diverge:

• Repository Reality governs the implemented system.

• Architectural Authority governs approved future change.

The two authorities SHALL remain synchronized through certified
repository convergence.
