# Repository Reality

Repository contents are the authoritative implementation state.

Architecture governs approved future change.

Repository convergence synchronizes both authorities.

---

# Artifact-First Principle

Implementation SHALL be produced as complete candidate artifacts.

Repository mutation SHALL occur only through certified repository
convergence.

Conversation-generated repository mutation is prohibited.

---

# Execution Package Standard

Repository mutation SHALL originate only from certified execution
packages.

Execution packages SHALL contain only the artifacts required for
repository convergence.

Execution artifacts SHALL remain isolated from the repository unless
explicitly approved by Architecture Authority.

---

# Execution Brief Standard

Every implementation work package SHALL begin with an approved
Execution Brief containing:

• Brief ID
• Architecture Approval
• Implementation Authority
• Candidate Root
• Repository Target
• Mission
• Deliverable
• Acceptance Criteria
• Stop Condition

---

# Phase 5 Freeze Exit Criteria

The baseline SHALL remain unchanged unless one or more of the following
conditions is verified:

1. a verified implementation defect exists;

2. repository integrity cannot be guaranteed;

3. an approved doctrine cannot be satisfied using the existing baseline.

Approved changes SHALL be:

• bounded;

• corrective;

• independently approved;

• unrelated architectural expansion is prohibited.

---

# Default Operating Mode

Phase 5 MVP Implementation

Architecture work is prohibited unless the Freeze Exit Criteria have
been satisfied.

Primary project activity SHALL be MVP implementation.
